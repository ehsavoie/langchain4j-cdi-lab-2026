Le contexte aujourd'hui

Vous avez peut-être vu :

LangChain4j + 🃏 Quarkus = 🤖 Chatbot intelligent

Mais en production, vous avez :

🐝 WildFly
🐠 Payara
🦅 Liberty
☀️ Helidon
🐡 GlassFish
🃏 Quarkus

Alors maintenant quoi ? 🤔

Vous avez peut-être vu de superbes démos avec LangChain4j sur Quarkus. Élégant, fluide, ça marchait du premier coup (enfin, presque). Mais soyons honnêtes : combien d'entre vous ont WildFly en production ? (levez la main !) Payara ? Liberty ? ... GlassFish ? (moment de silence) Bonne nouvelle : vous n'avez pas besoin de tout démolir et remplacer votre stack pour faire de l'IA. C'est exactement pourquoi nous avons créé langchain4j-cdi.

La réponse :

@RegisterAIService

CDI fait le pont entre LangChain4j et votre serveur

« Injectez un agent IA aussi naturellement qu'un EntityManager »

La réponse tient en une annotation : @RegisterAIService. CDI injecte déjà tout dans vos applications. EntityManager, DataSource, clients REST... alors pourquoi pas un agent IA ? C'est exactement ce que nous avons construit. Une extension CDI qui transforme une interface annotée en bean injectable, câblé à LangChain4j sous le capot. Mêmes patterns que vous connaissez déjà, capacités toutes nouvelles.

Programme de l'atelier

⚙️ Configuration et architecture — Prérequis, vue d'ensemble de la stack, internes de l'extension CDI
💬 Exercice 1 — Agent IA injectable — @RegisterAIService, streaming, vision multimodale
🛡️ Exercice 2 — Résilience et observabilité — Mémoire, RAG, outils, @Retry, @CircuitBreaker, OpenTelemetry
🔌 Exercice 3 — Intégration MCP — Model Context Protocol, fournisseurs d'outils externes
🚧 Exercice 4 — Guardrails — validation d'entrée/sortie, filtrage, sécurité des prompts
🤝 Exercice 5 — Agent to Agent (A2A) — orchestration multi-agents, protocole A2A, Java/Jakarta SDK
🚀 Conclusion et suite — roadmap, Q&R

Chaque exercice a un squelette base/ et une référence solution/ — travaillez à votre rythme.

Voici notre plan de bataille pour les prochaines heures. Nous commençons par une vérification rapide de la configuration et un aperçu théorique du fonctionnement de l'extension CDI sous le capot — le territoire d'Emmanuel. Ensuite cinq exercices pratiques, chacun construisant sur le précédent. L'exercice 1 est votre premier agent IA avec une seule annotation. L'exercice 2 ajoute la résilience et l'observabilité. L'exercice 3 connecte votre agent au monde extérieur via MCP. L'exercice 4 met en place des guardrails pour sécuriser les échanges. L'exercice 5 fait dialoguer plusieurs agents entre eux via le protocole A2A. Chaque exercice a un squelette base/ avec des TODOs et une solution/ pour quand vous êtes bloqué. Pas de jugement — la solution est là pour une raison. Allons-y !
